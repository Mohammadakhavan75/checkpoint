`AI-2026-05-10-005`

## Summary

Added the React web app with protected auth routes, compact foldable side rail, Today Start Ritual, Stop Ritual, Life Index, Parking, Settings, component tests, and Playwright E2E spec.

## Motivation

The MVP requires a minimalist ADHD-friendly UI that opens to Today and avoids dashboards, analytics, gamification, and unnecessary lists unless disclosed.

## Behavior before

No frontend application existed.

## Behavior after

Users can authenticate, create a first mission, resume from Today, leave checkpoints, view/activate/park missions, manage parking items, and update settings.

## Design decisions

- Route `/` directly to `/today` after auth.
- Keep navigation compact by default and persist it through preferences.
- Hide extended mission details behind `Show more`.
- Keep Stop Ritual limited to five fields.
- Use lucide icons for icon buttons and rail navigation.

## Alternatives considered

### Dashboard-first home

Rejected because the product rule requires opening to the Start Ritual.

### Rich editor/checklist UI

Rejected because the MVP optimizes for minimal cognitive load.
