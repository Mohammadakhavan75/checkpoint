# AI Change Record: Web App Dark Mode

## Change ID

`AI-2026-05-10-006`

## Task ID

`TASK-001`

## Summary

Added a user-triggered dark mode toggle to the web application. Introduced a `ThemeProvider` context to manage user preferences via `localStorage` and default to the system scheme via `window.matchMedia`.

## Motivation

To follow industry standards for user experience, offering a dark mode setting.

## Behavior before

The web app only supported a single light theme.

## Behavior after

Users can click the toggle in the AppShell header to switch between light and dark modes. The preference is persisted locally. Dark mode gracefully updates CSS variables for all visual components.

## Files changed

- `web/src/styles/app.css`
- `web/src/components/AppShell.tsx`
- `web/src/main.tsx`
- `web/src/lib/theme.tsx`
- `web/src/test-setup.ts`
- `web/src/components/AppShell.test.tsx`

## Design decisions

- Store the selected theme locally (`checkpoint-theme` in `localStorage`) rather than in backend preferences to keep the feature lightweight and presentation-focused.
- Refactored hardcoded colors in `app.css` to use CSS variables to allow seamless overriding in `.dark`.

## Alternatives considered

### Persisting to identity-service preferences
Rejected because adding dark mode solely via CSS and localStorage accomplishes the same goal immediately without requiring API contract modifications.

## Invariants preserved

- No behavior changes in existing components apart from rendering new visual styles based on class changes.
