# AI Change Record: Identity Service

## Change ID

`AI-2026-05-10-002`

## Task ID

`TASK-0001`

## Summary

Added the identity service with signup, login, refresh, logout, user lookup, and per-user preferences.

## Motivation

Checkpoint requires multi-user email/password auth, HttpOnly cookie-compatible token payloads, Redis-backed refresh sessions, and per-user active-limit/nav preferences.

## Behavior before

No users, preferences, password auth, or refresh sessions existed.

## Behavior after

Users can sign up and log in, receive access/refresh tokens through the gateway, rotate refresh sessions, log out, and read/update preferences.

## Design decisions

- Hash passwords with bcrypt through Passlib.
- Store refresh sessions server-side in Redis for Docker and an in-memory store for tests.
- Default `nav_collapsed` to true and `active_limit` to one.

## Alternatives considered

### Local storage auth

Rejected because HttpOnly cookie auth and multi-user isolation were required.

### Stateless refresh tokens

Rejected because Redis-backed refresh session invalidation was required.

## Invariants reviewed

- `docs/invariants/security.md`

## Invariants preserved

- Passwords are never stored or returned in plaintext.
- Invalid credentials and invalid refresh sessions fail closed.
- Default preferences are created server-side.
