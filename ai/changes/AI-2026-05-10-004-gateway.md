# AI Change Record: API Gateway

## Change ID

`AI-2026-05-10-004`

## Task ID

`TASK-0001`

## Summary

Added the FastAPI gateway exposing the public `/api` contract with HttpOnly auth cookies and proxy routes to the internal services.

## Motivation

The web app needs one public API that hides internal service topology, sets auth cookies, validates access tokens, and coordinates active-limit preference enforcement.

## Behavior before

No public `/api` interface existed.

## Behavior after

The gateway supports auth, preferences, domains, missions, activation/parking, Today, checkpoints, and parking item routes.

## Files changed

- `services/api-gateway/app/main.py`
- `services/api-gateway/app/security.py`

## Design decisions

- Use HttpOnly `access_token` and `refresh_token` cookies.
- Decode access tokens at the gateway and forward `X-User-Id` to checkpoint service.
- Use explicit HTTPX timeouts for upstream calls.
- Disable FastAPI response-model inference on proxy routes.

## Alternatives considered

### Browser-managed bearer tokens

Rejected because HttpOnly cookie auth was required.

### Direct frontend calls to each service

Rejected because the MVP exposes a single `/api` public interface.

## Invariants preserved

- Protected routes fail closed without a valid cookie.
- Upstream requests are bounded by explicit timeouts.
- The public API shape is centralized under `/api`.
