# AI Change Record: Docker Infrastructure

## Change ID

`AI-2026-05-10-001`

## Task ID

`TASK-0001`

## Summary

Added local Docker Compose infrastructure, environment defaults, ignore rules, service Dockerfiles, and repository README for the Checkpoint MVP.

## Motivation

The MVP must run as a Dockerized microservice stack with web, gateway, internal services, Postgres, and Redis.

## Behavior before

There was no runnable Checkpoint application stack.

## Behavior after

`docker compose up --build` starts the web app, gateway, identity service, checkpoint service, Postgres, and Redis with health checks.

- `.env.example`

- `README.md`
- `docker-compose.yml`
- `services/*/Dockerfile`
- `services/*/.dockerignore`
- `services/*/requirements.txt`
- `web/Dockerfile`
- `web/.dockerignore`
- `web/package.json`

## Design decisions

- Use Docker Compose for local-first operation.
- Expose only web and gateway to the host.
- Use Docker health checks so service startup order is explicit.

## Alternatives considered

### Single FastAPI app

Rejected because the requested architecture specified separate gateway, identity, and checkpoint services.

### Cloud deployment first

Rejected because the MVP assumption is local Docker-first.
